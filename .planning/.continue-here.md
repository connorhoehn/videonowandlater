<context>
The processing history log implementation adds proper event logging to track when each video processing step (MediaConvert conversion, Transcribe transcription, AI summary) starts, completes, or fails. This replaces the current approach where the frontend infers a timeline from status fields.

The implementation follows the plan from the previous planning session. The approach uses the existing DynamoDB single-table design, storing events as separate items under each session. Events are created in a non-blocking manner to ensure processing continues even if event logging fails.

Key benefits:
- Real timestamps for each processing step
- Detailed error information when steps fail
- Complete audit trail of video processing
- Backward compatible with existing sessions

The work is being done systematically: first adding domain types, then repository methods, then updating handlers to emit events, and finally updating the frontend to display them.
</context>
