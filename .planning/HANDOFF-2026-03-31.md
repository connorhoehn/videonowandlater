## Remaining audit items (not addressed)
- **DynamoDB scans** in `stream-started`, `stream-ended`, `recording-started`, `recording-ended`, `findSessionByStageArn` — need GSI on `claimedResources.channel` / `.stage` for O(1) lookups. Currently O(n) scans — acceptable at current scale but will need GSI before hitting ~1000 sessions.
- **Pagination** in `getRecentActivity` / `getRecentRecordings` — only returns first page of scan results.
- **Large transcript memory** in `transcribe-completed.ts` — loads entire Transcribe JSON into memory. For >30min videos, consider streaming.
- **Remaining `console.log` calls** in ~15 API handlers (non-pipeline) — lower priority, these are API Gateway handlers not event-driven pipeline.
- **Missing optimistic locking** on `updateRecordingMetadata` — version field is incremented but not checked. Low risk since concurrent writes to the same session's recording metadata are rare.
