### Critical Integration Gaps

#### 1. Frontend Routing Missing (CRITICAL)

**Affected Requirements:** BCAST-01, BCAST-02, BCAST-03, BCAST-05, BCAST-06, CHAT-01, CHAT-02, CHAT-03

**Issue:** BroadcastPage and ViewerPage components exist but are never imported or routed in App.tsx.

**Files:**
- `web/src/App.tsx` (missing routes for `/broadcast/:sessionId` and `/viewer/:sessionId`)
- `web/src/features/broadcast/BroadcastPage.tsx` (orphaned component)
- `web/src/features/viewer/ViewerPage.tsx` (orphaned component)

**Impact:** Users cannot access broadcast or viewer functionality despite backend APIs and frontend components being complete.

**Resolution:** Add routes to App.tsx:
```tsx
<Route path="/broadcast/:sessionId" element={<BroadcastPage />} />
<Route path="/viewer/:sessionId" element={<ViewerPage />} />
```

#### 2. Session Creation UI Missing (CRITICAL)

**Affected Requirements:** SESS-01, BCAST-01, POOL-05

**Issue:** Backend provides `POST /sessions` endpoint to create sessions, but frontend HomePage has placeholder text "Future phases will add session creation and management here" with no actual implementation.

**Files:**
- `infra/lib/stacks/api-stack.ts` lines 74-89 (API exists)
- `web/src/pages/HomePage.tsx` (no UI)

**Impact:** Users cannot create broadcasts or hangouts. E2E flow broken at step 1.

**Resolution:** Add session creation UI to HomePage with "Create Broadcast" button that calls POST /sessions API.

#### 3. API Configuration Mismatch (MAJOR)

**Affected Requirements:** CHAT-01, CHAT-03 (runtime failures), all other API-dependent requirements (latent risk)

**Issue:** Deploy script generates `web/public/aws-config.json` with `apiUrl` property, but frontend components use inconsistent sources:
- BroadcastPage/ViewerPage: `import.meta.env.VITE_API_URL` (environment variable, never set)
- useChatRoom/ChatMessagesProvider: `window.APP_CONFIG?.apiBaseUrl` (undefined, wrong property name)

**Expected:** All components should use `getConfig()` from `web/src/config/aws-config.ts` to access `config.apiUrl`.

**Impact:** API calls will fail with default hardcoded URLs instead of using deployed API Gateway endpoint.

**Resolution:** Standardize all components to use `getConfig().apiUrl` from aws-config.ts.
