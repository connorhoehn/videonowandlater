- `-i ${options.videoFile}`
- `-c:v libvpx-vp8 -b:v 2000k` (WebRTC requires VP8 or H.264)

Implementation:
1. Import Session, SessionType, SessionStatus from ../../domain/session
2. Import uuid from 'uuid'
3. Parse count from options.count (parseInt)
4. Loop from 0 to count:
   - Generate sessionId via v4()
   - Alternate sessionType: i % 2 === 0 ? BROADCAST : HANGOUT
   - Create timestamps: createdAt (now - (count - i) * 3600000 for 1hr intervals), startedAt (createdAt + 30s), endedAt (startedAt + 30min)
   - Build Session object with userId: `test-user-${i % 3}`, status: ENDED, claimedResources with placeholder ARNs
   - Call DynamoDB PutCommand directly (or reuse session-repository if it exports createSession helper)

   - Console.log progress: "Created BROADCAST session: abc-123"

Create backend/src/cli/commands/seed-chat.ts with async function seedChat(sessionId: string, options: { count: string }).

Implementation:
1. Import ChatMessage, calculateSessionRelativeTime from ../../domain/chat-message
2. Import BatchWriteCommand from @aws-sdk/lib-dynamodb
3. Parse count from options
4. Fetch session via getSessionById to get sessionStartedAt
5. Generate chat messages array:
   - messageId: `msg-${i}`
   - sessionId: from argument
   - senderId: `user-${i % 3}` (rotate through 3 users)
   - content: `Test message ${i}`
   - sentAt: sessionStartedAt + (i * 5000) for 5-second intervals
   - sessionRelativeTime: calculateSessionRelativeTime(sessionStartedAt, sentAt)
   - senderAttributes: { displayName: `User ${i % 3}` }
6. Chunk messages into groups of 25 (DynamoDB BatchWrite limit)
7. For each chunk, call BatchWriteCommand with PK: `MESSAGE#${sessionId}`, SK: `${sentAt}#${messageId}`, entityType: 'MESSAGE'
8. Console.log progress and final count

Create backend/src/cli/commands/seed-reactions.ts with async function seedReactions(sessionId: string, options: { count: string, replay?: boolean }).

Implementation:
1. Import Reaction, ReactionType from ../../domain/reaction
2. Import calculateShardKey from ../../repositories/reaction-repository (or inline: hash-based sharding UTF-8 sum mod 100)
3. Import BatchWriteCommand from @aws-sdk/lib-dynamodb
4. Parse count from options
5. Fetch session to get recordingDuration (only seed reactions within [0, recordingDuration])
6. Generate reactions array:
   - reactionId: uuid()
   - sessionId: from argument
   - userId: `user-${Math.floor(Math.random() * 10)}` (10 random users)
   - reactionType: random from [heart, fire, clap, laugh, surprised]
   - sessionRelativeTime: Math.floor(Math.random() * recordingDuration * 1000) (random within duration)
   - shardKey: calculateShardKey(reactionId) (mod 100)
   - isReplayReaction: options.replay || false
   - createdAt: new Date().toISOString()
7. Chunk into groups of 25
8. For each chunk, BatchWriteCommand with:
   - PK: `REACTION#${sessionId}#SHARD${shardKey.toString().padStart(2, '0')}`
   - SK: `${sessionRelativeTime.toString().padStart(10, '0')}#${reactionId}`
   - GSI2PK: `REACTION#${sessionId}`
   - GSI2SK: `${sessionRelativeTime.toString().padStart(10, '0')}#${reactionId}`
   - entityType: 'REACTION'
9. Console.log progress and final count with shard distribution
