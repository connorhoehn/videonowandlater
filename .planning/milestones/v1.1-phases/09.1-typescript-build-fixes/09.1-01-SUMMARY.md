# Phase 09.1 Plan 01: TypeScript Build Fixes — Hangout Hooks Summary

Fixed six TypeScript compilation errors across two Phase 8 hangout hooks by correcting amazon-ivs-web-broadcast Stage SDK import usage, event names, return types, and a cross-environment timer type.

## Changes Made

### web/src/features/hangout/useHangout.ts

**Fix 1a — Import line (line 7):**
```diff
-import { Stage, StageStrategy, SubscribeType } from 'amazon-ivs-web-broadcast';
+import { Stage, type StageStrategy, SubscribeType, StageEvents, LocalStageStream } from 'amazon-ivs-web-broadcast';
```
Added `StageEvents` (needed for event listeners) and `LocalStageStream` (needed for return type). Applied inline `type` modifier to `StageStrategy` (required for isolatedModules).

**Fix 1b — Remove unused `stage` state:**
```diff
-const [stage, setStage] = useState<Stage | null>(null);
```
Also removed the corresponding `setStage(stageInstance)` call. The state was declared but never consumed in the return value, causing TS6133 "declared but never read".

**Fix 1c — stageStreamsToPublish return type (line 54):**
```diff
-stageStreamsToPublish: () => SubscribeType.AUDIO_VIDEO,
+stageStreamsToPublish: (): LocalStageStream[] => [],
```
The `StageStrategy` interface requires `stageStreamsToPublish()` to return `LocalStageStream[]`, not `SubscribeType`. Returning an empty array is the correct minimal implementation.

**Fix 1d — Event names (3 occurrences):**
```diff
-stageInstance.on(Stage.Events.STAGE_PARTICIPANT_JOINED, ...
+stageInstance.on(StageEvents.STAGE_PARTICIPANT_JOINED, ...

-stageInstance.on(Stage.Events.STAGE_PARTICIPANT_LEFT, ...
+stageInstance.on(StageEvents.STAGE_PARTICIPANT_LEFT, ...

-StageEvents.STAGE_PARTICIPANT_STREAMS_CHANGED,
+StageEvents.STAGE_PARTICIPANT_STREAMS_ADDED,
```
`Stage.Events` does not exist on the SDK class — `StageEvents` is a separate named export. Also corrected the event name: `STREAMS_CHANGED` -> `STREAMS_ADDED` (the actual SDK enum value).

**Fix 1e — stage.leave() synchronous call:**
```diff
-stageInstance.leave().catch(console.error);
+stageInstance.leave();
```
`Stage.leave()` returns `void`, not a Promise. Calling `.catch()` on void causes TS2339.

**Auto-fix (Rule 1 — Bug) — STAGE_PARTICIPANT_STREAMS_ADDED callback type:**
```diff
-(participant: any, streams: MediaStream[]) => {
+(participant: any, streams: any[]) => {
```
Found during Task 3 full build verification. The SDK emits `StageStream<never>[]` for this event, not `MediaStream[]`. Changed to `any[]` consistent with `participant: any` pattern. Resolves TS2345.

### web/src/features/hangout/useActiveSpeaker.ts

**Fix — intervalId type (line 29):**
```diff
-let intervalId: NodeJS.Timeout;
+let intervalId: ReturnType<typeof setInterval>;
```
The web tsconfig uses `types: ["vite/client"]` — `@types/node` is unavailable, so `NodeJS.Timeout` does not exist. `ReturnType<typeof setInterval>` is the canonical cross-environment solution resolving to `number` in browsers.
