## Decisions Made

- NODE_OPTIONS env var approach over `jest.config.js extensionsToTreatAsEsm`: The `jest.config.js` uses CommonJS `module.exports` format. The ESM fix is needed at the Node runtime level, not the Jest transform level. AWS SDK v3 uses dynamic `import()` internally which requires the experimental flag.
- Did NOT add `"type": "module"` to `backend/package.json`: The backend uses NodeNext module resolution via tsconfig; adding `"type":"module"` would break CommonJS interop in Lambda build output.
- Did NOT modify `jest.config.js`: No changes needed there — the runtime flag approach is sufficient.
