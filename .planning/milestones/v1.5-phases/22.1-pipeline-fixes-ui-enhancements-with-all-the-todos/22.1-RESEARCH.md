### Pattern 1: CDK Custom Resource for Cleanup
**What:** Lambda-backed custom resource that runs on CloudFormation DELETE events
**When to use:** When AWS resources have dependencies that block standard deletion
**Example:**
```typescript
// Source: AWS CDK best practices
const cleanupFunction = new lambda.Function(this, 'CleanupFunction', {
  runtime: lambda.Runtime.NODEJS_20_X,
  handler: 'index.handler',
  code: lambda.Code.fromInline(`
    exports.handler = async (event) => {
      if (event.RequestType === 'Delete') {
        // Detach all recording configurations from channels
        const ivs = new AWS.IVS();
        const channels = await ivs.listChannels().promise();
        for (const channel of channels.Channels) {
          if (channel.recordingConfigurationArn) {
            await ivs.updateChannel({
              arn: channel.arn,
              recordingConfigurationArn: ''
            }).promise();
          }
        }
      }
      return { PhysicalResourceId: 'cleanup-complete' };
    };
  `)
});

new customResources.Provider(this, 'CleanupProvider', {
  onEventHandler: cleanupFunction
});
```

### Pattern 2: Activity Card Variants
**What:** Polymorphic rendering based on sessionType
**When to use:** Different session types need different UI representations
**Example:**
```typescript
// Source: React patterns for conditional rendering
const ActivityFeed = ({ sessions }) => {
  return sessions.map(session => {
    switch(session.sessionType) {
      case 'BROADCAST':
        return <BroadcastActivityCard session={session} />;
      case 'HANGOUT':
        return <HangoutActivityCard session={session} />;
      case 'UPLOAD':
        return <UploadActivityCard session={session} />;
      default:
        return null;
    }
  });
};
```

## Common Pitfalls

### Pitfall 1: EventBridge Rule Filtering
**What goes wrong:** MediaConvert jobs complete but handlers don't trigger
**Why it happens:** Missing or incorrect detail filters in EventBridge rules
**How to avoid:** Always include userMetadata filters when jobs are tagged
**Warning signs:** Jobs show COMPLETE in MediaConvert console but DynamoDB shows pending

### Pitfall 2: IVS Resource Dependencies
**What goes wrong:** Stack deletion fails with ConflictException
**Why it happens:** RecordingConfiguration attached to channels blocks deletion
**How to avoid:** Implement cleanup custom resource before RecordingConfiguration
**Warning signs:** Stack deletion hangs or requires multiple retries

### Pitfall 3: Missing Session Type Handling
**What goes wrong:** Upload sessions don't appear in activity feed
**Why it happens:** ActivityFeed only handles BROADCAST and HANGOUT types
**How to avoid:** Add explicit case for UPLOAD sessionType
**Warning signs:** Uploads redirect to replay but don't show in dashboard
