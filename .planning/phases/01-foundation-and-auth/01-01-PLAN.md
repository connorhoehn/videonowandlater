<task type="auto">
  <name>Task 2: Create CDK Auth and Monitoring stacks</name>
  <files>infra/bin/app.ts, infra/lib/stacks/auth-stack.ts, infra/lib/stacks/monitoring-stack.ts</files>
  <action>
1. **`infra/lib/stacks/auth-stack.ts`**: Create AuthStack extending cdk.Stack.
   - Define `UserPool` with:
     - `userPoolName: 'vnl-user-pool'`
     - `selfSignUpEnabled: true`
     - `signInAliases: { username: true }`
     - `signInCaseSensitive: false`
     - `passwordPolicy: { minLength: 8, requireLowercase: true, requireUppercase: true, requireDigits: true, requireSymbols: false }`
     - `removalPolicy: RemovalPolicy.DESTROY` (INFRA-02: clean teardown)
   - Define `UserPoolClient` with:
     - `authFlows: { userPassword: true, userSrp: true, adminUserPassword: true }` (adminUserPassword needed for DEV-02 in Plan 02)
     - `preventUserExistenceErrors: true`
     - `accessTokenValidity: Duration.hours(1)`
     - `refreshTokenValidity: Duration.days(30)`
   - Expose `userPool` and `userPoolClient` as public readonly properties (for cross-stack reference by ApiStack in Plan 02)
   - Add CfnOutputs: `UserPoolId`, `UserPoolClientId`, `CognitoRegion` (using `this.region`)

2. **`infra/lib/stacks/monitoring-stack.ts`**: Create MonitoringStack extending cdk.Stack.
   - Create SNS Topic `BillingAlarmTopic` with displayName `'VNL Billing Alerts'`
   - Loop over thresholds `[10, 50, 100]`:
     - Create CloudWatch Metric on `AWS/Billing` namespace, `EstimatedCharges` metricName, `Maximum` statistic, `Duration.hours(6)` period, `dimensionsMap: { Currency: 'USD' }`
     - Create Alarm with the threshold, `evaluationPeriods: 1`, `GREATER_THAN_THRESHOLD` comparison, `treatMissingData: NOT_BREACHING`, alarmName `vnl-billing-${threshold}`
     - Add SNS action to alarm
   - Add CfnOutput for `BillingAlarmTopicArn`

3. **`infra/bin/app.ts`**: CDK app entry point.
   - Import AuthStack and MonitoringStack
   - Create CDK App
   - Set `env: { region: 'us-east-1' }` (REQUIRED: billing alarms only work in us-east-1, per research pitfall #1)
   - Instantiate `AuthStack` as `'VNL-Auth'`
   - Instantiate `MonitoringStack` as `'VNL-Monitoring'`
   - Do NOT instantiate ApiStack yet (created in Plan 02)
   - Export `authStack` for later use when ApiStack is added

4. Run `npx cdk synth` to verify templates generate without errors.

Auth stack exposes userPool and userPoolClient properties for cross-stack reference.</done>
