<task type="auto">
  <name>Task 2: Create deploy/destroy scripts and developer CLI tools</name>
  <files>scripts/deploy.sh, scripts/destroy.sh, scripts/create-user.sh, scripts/list-users.sh, scripts/delete-user.sh, scripts/get-token.sh</files>
  <action>
Create `scripts/` directory, then write each script:

1. **`scripts/deploy.sh`**:
   - `set -euo pipefail`
   - Run `npx cdk deploy --all --require-approval never --outputs-file cdk-outputs.json`
   - Create `web/public/` directory if it doesn't exist (`mkdir -p web/public`)
   - Transform CDK outputs with jq: extract `userPoolId` from `."VNL-Auth".UserPoolId`, `userPoolClientId` from `."VNL-Auth".UserPoolClientId`, `region` from `."VNL-Auth".CognitoRegion`, `apiUrl` from `."VNL-Api".ApiUrl` -- write to `web/public/aws-config.json`
   - Print success message

2. **`scripts/destroy.sh`**:
   - `set -euo pipefail`
   - Run `npx cdk destroy --all --force`
   - Remove `cdk-outputs.json` and `web/public/aws-config.json` (use `rm -f`)
   - Print success message

3. **`scripts/create-user.sh`**:
   - Usage: `./scripts/create-user.sh <username> <password>`
   - Read `USER_POOL_ID` from `cdk-outputs.json` using `jq -r '."VNL-Auth".UserPoolId'`
   - If cdk-outputs.json missing, print error and exit 1
   - Run `aws cognito-idp admin-create-user` with `--message-action SUPPRESS`
   - Then run `aws cognito-idp admin-set-user-password --permanent` to bypass FORCE_CHANGE_PASSWORD (per research pitfall #7)
   - Print confirmation

4. **`scripts/list-users.sh`**:
   - Read `USER_POOL_ID` from `cdk-outputs.json`
   - Run `aws cognito-idp list-users` and format with `jq` to show Username and UserStatus
   - Print results in readable format

5. **`scripts/delete-user.sh`**:
   - Usage: `./scripts/delete-user.sh <username>`
   - Read `USER_POOL_ID` from `cdk-outputs.json`
   - Run `aws cognito-idp admin-delete-user`
   - Print confirmation

6. **`scripts/get-token.sh`**:
   - Usage: `./scripts/get-token.sh <username> <password>`
   - Read `USER_POOL_ID` and `CLIENT_ID` (UserPoolClientId) from `cdk-outputs.json`
   - Run `aws cognito-idp admin-initiate-auth` with `ADMIN_USER_PASSWORD_AUTH` flow
   - Output AccessToken, IdToken, RefreshToken, ExpiresIn using `jq`
   - Print formatted result

7. Make all scripts executable: `chmod +x scripts/*.sh`
  </action>
