  <files>web/package.json, web/tsconfig.json, web/vite.config.ts, web/index.html, web/src/main.tsx, web/src/config/aws-config.ts, web/src/auth/amplify.ts, web/src/auth/AuthContext.tsx, web/src/auth/useAuth.ts</files>
  <action>
1. **Scaffold the Vite React app**: Run `npm create vite@latest web -- --template react-ts` from the repo root. Then `cd web && npm install aws-amplify react-router-dom`. Add `@types/react @types/react-dom` as devDependencies if not already present.

2. **Update root `package.json`**: Ensure the `workspaces` array includes `"web"`. Run `npm install` from root to link.

3. **`web/vite.config.ts`**: Standard Vite config with React plugin. No special env vars (config loaded at runtime, not build time -- per research anti-pattern #5).

4. **`web/src/config/aws-config.ts`**: Runtime config loader.
   - Define `AwsConfig` interface: `{ userPoolId: string, userPoolClientId: string, region: string, apiUrl: string }`
   - `loadConfig()`: async function that fetches `/aws-config.json`, returns `AwsConfig | null`. On fetch failure or non-ok response, return null (stack not deployed).
   - `getConfig()`: synchronous getter returning the cached config or null.
   - Store config in module-level variable.

5. **`web/src/auth/amplify.ts`**: Amplify auth wrapper functions.
   - `configureAuth(config: AwsConfig)`: Call `Amplify.configure({ Auth: { Cognito: { userPoolId, userPoolClientId } } })`. Import from `'aws-amplify'`.
   - `handleSignUp(username, password)`: Call `signUp({ username, password })` from `'aws-amplify/auth'`. Return the result.
   - `handleSignIn(username, password)`: Call `signIn({ username, password })` from `'aws-amplify/auth'`. Return the result.
   - `handleSignOut()`: Call `signOut()` from `'aws-amplify/auth'`.
   - `checkSession()`: Try `getCurrentUser()` + `fetchAuthSession()` from `'aws-amplify/auth'`. Return `{ username, tokens }` on success, `null` on failure (not authenticated).

6. **`web/src/auth/AuthContext.tsx`**: React context for auth state.
   - Define `AuthState` type: `{ user: { username: string } | null, isLoading: boolean, isAuthenticated: boolean, signIn: (username: string, password: string) => Promise<void>, signUp: (username: string, password: string) => Promise<void>, signOut: () => Promise<void> }`
   - Create `AuthContext` with `createContext`
   - `AuthProvider` component:
     - State: `user`, `isLoading` (starts true)
     - On mount: call `checkSession()`. If session exists, set user. Set isLoading false.
     - `signIn`: call `handleSignIn`, then `checkSession` to get user, update state.
     - `signUp`: call `handleSignUp`, then auto-sign-in (call `handleSignIn` with same credentials), then `checkSession`.
     - `signOut`: call `handleSignOut`, set user to null.
     - Provide `{ user, isLoading, isAuthenticated: !!user, signIn, signUp, signOut }` via context.

7. **`web/src/auth/useAuth.ts`**: Simple hook that calls `useContext(AuthContext)` and throws if used outside provider.

8. Run `cd web && npm run build` to verify TypeScript compiles.
