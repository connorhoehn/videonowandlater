## Key Context

**transcribe-completed:**
- Receives AWS Transcribe job completion events via EventBridge
- Event detail includes: TranscriptionJob with TranscriptionJobStatus, TranscriptionJobName, Results
- Needs to validate job status enum and Results structure

**store-summary:**
- Receives trigger event with sessionId and transcriptS3Uri
- Both fields required (permanent failure if missing)
- Used to invoke Bedrock for AI summary generation

**on-mediaconvert-complete:**
- Receives MediaConvert job completion details
- Similar to transcode-completed but lighter payload
- Validates jobName and status

**Validation pattern (from Plan 02):**
- Parse JSON from record.body
- Validate EventBridge envelope
- Validate detail schema
- On failure: log + add to failures[]
- On success: call processEvent with typed detail

**All three handlers are traced (X-Ray active from Phase 36), so validation failures will be visible in traces.**
</context>
