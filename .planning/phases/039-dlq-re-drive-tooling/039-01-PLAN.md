<objective>
Build a complete CLI toolkit for inspecting, re-driving, and managing messages in the 5 pipeline DLQs, replacing manual AWS console operations with automated developer commands.

Purpose: Operators need a programmatic way to understand what failed in the pipeline, fix the root cause, and safely re-drive messages without touching the AWS console. X-Ray tracing (Phase 36) makes re-driven messages observable end-to-end, enabling confident remediation workflows.

Output: Four new CLI commands (dlq-list, dlq-redrive, dlq-purge, dlq-health) with comprehensive test coverage, fully integrated into the existing vnl-cli tool.
</objective>
