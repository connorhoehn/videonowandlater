  <name>Task 1: Create Recording Infrastructure in SessionStack</name>
  <files>infra/lib/stacks/session-stack.ts</files>
  <action>
Add recording infrastructure to SessionStack CDK construct:

1. **S3 Bucket for recordings:**
   - Name: `vnl-recordings-${stage}` (use existing stage pattern from stack)
   - Block all public access
   - S3-managed encryption
   - RemovalPolicy.DESTROY with autoDeleteObjects (consistent with existing resources)
   - Region: us-east-1 (same as existing stacks per PROJECT.md)

2. **CloudFront Distribution with OAC:**
   - Create CfnOriginAccessControl with s3 origin type, sigv4 signing
   - Create Distribution with S3Origin using OAC
   - ViewerProtocolPolicy: REDIRECT_TO_HTTPS
   - CachePolicy: CACHING_OPTIMIZED (recordings are immutable)
   - Add S3 bucket policy granting CloudFront GetObject access (condition on distribution ARN)

3. **IVS RecordingConfiguration (L1 constructs):**
   - Create CfnStorageConfiguration pointing to S3 bucket
   - Create CfnRecordingConfiguration with:
     - destinationConfiguration.s3.bucketName: recordingsBucket.bucketName
     - thumbnailConfiguration: recordingMode='INTERVAL', targetIntervalSeconds=10, resolution='HD'
     - renditionConfiguration: renditions=['ALL']
   - Export ARN via CfnOutput: 'vnl-recording-config-arn'
   - Export CloudFront domain via CfnOutput: 'vnl-recordings-domain'

4. **EventBridge Rules for Recording Lifecycle:**
   - Create Rule for 'Recording Start' events (source: aws.ivs, detailType: IVS Recording State Change, detail.event_name: Recording Start)
   - Create Rule for 'Recording End' events (same source/detailType, detail.event_name: Recording End)
   - Targets will be wired in Plan 05-02 (handlers created there)
   - Store rules as class properties for later Lambda target attachment

Use aws-cdk-lib imports (CfnRecordingConfiguration, CfnStorageConfiguration from aws-cdk-lib/aws-ivs). Follow existing SessionStack pattern (NodejsFunction, EventBridge Rule creation). Add necessary IAM permissions to existing Lambda roles for S3 read access to recording metadata files.

1. **Add RecordingStatus enum:**

   ```typescript
   export enum RecordingStatus {
     PENDING = 'pending',      // Session created but recording not started
     PROCESSING = 'processing', // Recording in progress or finalizing
     AVAILABLE = 'available',   // Recording complete and ready
     FAILED = 'failed',        // Recording encountered an error
   }
   ```

2. **Extend Session interface with optional recording fields:**
   - recordingS3Path?: string (S3 key prefix from IVS, e.g., "ivs/v1/123456/2026/3/2/...")
   - recordingDuration?: number (duration in milliseconds from IVS event)
   - thumbnailUrl?: string (CloudFront URL for first thumbnail)
   - recordingHlsUrl?: string (CloudFront URL for HLS manifest)
   - recordingStatus?: RecordingStatus (lifecycle state)

All fields optional (sessions don't have recordings until stream starts). Add as flat fields on Session interface, not nested object (per CONTEXT.md decision). Use same naming convention as existing fields (camelCase).

Export RecordingStatus enum for use in handlers and repository.
