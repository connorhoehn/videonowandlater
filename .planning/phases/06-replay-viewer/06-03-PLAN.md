  <action>
    Create React hook that filters chat messages based on video playback position.

    **useSynchronizedChat Hook (web/src/features/replay/useSynchronizedChat.ts):**
    Export `export function useSynchronizedChat(allMessages: ChatMessage[], currentSyncTime: number): ChatMessage[]`

    Implementation:
    - Use useMemo with dependencies [allMessages, currentSyncTime]
    - Filter logic:
      ```typescript
      return useMemo(() => {
        if (currentSyncTime === 0) return []; // No playback started yet

        return allMessages.filter(
          msg => msg.sessionRelativeTime !== undefined &&
                 msg.sessionRelativeTime <= currentSyncTime
        );
      }, [allMessages, currentSyncTime]);
      ```

    **Key insight from RESEARCH.md:**
    - syncTime is UTC milliseconds from IVS Player getSyncTime (wall-clock time)
    - sessionRelativeTime is milliseconds since stream start (stored on ChatMessage in Phase 4)
    - Comparison `sessionRelativeTime <= currentSyncTime` shows messages up to current playback position
    - Filtering in useMemo prevents unnecessary re-renders (SYNC_TIME_UPDATE fires 1Hz)

    **Why useMemo:** SYNC_TIME_UPDATE event fires every second during playback. Re-filtering entire message array (potentially 500+ messages) on every event without memoization causes performance issues. useMemo caches result until dependencies change.

    Pattern follows existing hooks in web/src/features/ (e.g., usePlayer.ts, useBroadcast.ts).
