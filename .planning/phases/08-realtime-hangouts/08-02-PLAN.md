```typescript
import { Stage, StageStrategy, SubscribeType } from 'amazon-ivs-web-broadcast';

interface StageStrategy {
  stageStreamsToPublish(): SubscribeType;
  shouldPublishParticipant(): boolean;
  shouldSubscribeToParticipant(): SubscribeType;
}

class Stage {
  constructor(token: string, strategy: StageStrategy);
  join(): Promise<void>;
  leave(): Promise<void>;
  on(event: string, handler: Function): void;

  static Events = {
    STAGE_PARTICIPANT_JOINED: 'participantJoined',
    STAGE_PARTICIPANT_LEFT: 'participantLeft',
    STAGE_PARTICIPANT_STREAMS_CHANGED: 'participantStreamsChanged',
  };
}
```

    6. Cleanup: stage.leave() in useEffect return
    7. Error handling: try/catch around join flow, set error state

    Mirror the structure of useBroadcast.ts but adapt for Stage APIs instead of Channel APIs.

  <name>Task 2: Create useActiveSpeaker hook with Web Audio API for audio level monitoring</name>
  <files>web/src/features/hangout/useActiveSpeaker.ts</files>
  <action>
    Create React hook for active speaker detection (based on research Pattern 4):
    1. Hook signature: useActiveSpeaker({ participants, threshold?, smoothingTimeConstant? }): { activeSpeakerId: string | null }
    2. Parameters:
       - participants: Array&lt;{ participantId, streams }&gt;
       - threshold: number = -40 (dB threshold for "speaking" detection)
       - smoothingTimeConstant: number = 0.8 (frequency smoothing)
    3. State: activeSpeakerId: string | null
    4. In useEffect (dependencies: participants, threshold, smoothingTimeConstant):
       - Create Map&lt;participantId, AudioContext&gt; and Map&lt;participantId, AnalyserNode&gt;
       - For each participant with audio tracks:
         - Create new AudioContext()
         - Create analyser: audioContext.createAnalyser() with fftSize: 512, smoothingTimeConstant
         - Create source: audioContext.createMediaStreamSource(participant.streams[0])
         - Connect: source.connect(analyser)
       - Set interval (100ms) to poll audio levels:
         - For each analyser: getFloatTimeDomainData(dataArray), calculate RMS volume: sqrt(sum(dataArray[i]^2) / length)
         - Convert to dB: 20 * log10(rms)
         - Find loudest participant above threshold, update activeSpeakerId
       - Cleanup: clearInterval, close all AudioContexts
    5. Return { activeSpeakerId }

    This implements client-side active speaker detection without ML - sufficient for visual indicator (green border on loudest participant).
  </action>
