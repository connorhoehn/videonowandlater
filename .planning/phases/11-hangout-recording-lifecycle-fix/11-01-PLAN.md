<objective>
Fix Stage ARN detection and EventBridge routing so hangout composite recordings get metadata written and appear in the home feed.

Purpose: IVS RealTime Stage recording-end events are never reaching the recordingEndedFn Lambda (no CDK rule exists for them) and when they do reach it, the handler reads the wrong field for the ARN (`event.detail.channel_name` is undefined for Stage events — the ARN lives in `event.resources[0]`). This causes hangout recordings to silently fail metadata writes, blocking HANG-14, HANG-15, and HANG-16.

Output: CDK gains a StageRecordingEndRule; recording-ended.ts reads `event.resources[0]` and builds correct S3 paths for both event types; tests cover both shapes.
</objective>
