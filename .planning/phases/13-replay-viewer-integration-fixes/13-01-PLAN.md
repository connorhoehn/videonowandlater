**useReplayPlayer.ts — one edit:**

Change the SYNC_TIME_UPDATE event handler from:
```typescript
player.addEventListener(window.IVSPlayer.PlayerEventType.SYNC_TIME_UPDATE, (time: number) => {
  setSyncTime(time);
});
```
To:
```typescript
player.addEventListener(window.IVSPlayer.PlayerEventType.SYNC_TIME_UPDATE, () => {
  setSyncTime(player.getPosition() * 1000);
});
```

Rationale: `time` from SYNC_TIME_UPDATE is UTC epoch ms (~1.7 trillion), but `sessionRelativeTime` stored on chat messages and reactions is milliseconds elapsed since stream start (e.g., 5000 for a message 5 seconds in). `player.getPosition()` returns elapsed playback seconds since the stream's start — multiplying by 1000 converts to ms matching the `sessionRelativeTime` domain. This fixes both `useSynchronizedChat` and `useReactionSync` (both compare `sessionRelativeTime <= currentSyncTime`) and also fixes `ReactionTimeline` (`isHighlighted = currentTime >= bucketStartTime` where `bucketStartTime` is in relative ms).

Also update the comment on line 15 from `// UTC milliseconds from getSyncTime` to `// Elapsed playback milliseconds from player.getPosition()`.
