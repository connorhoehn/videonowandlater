# Plan 13-01: Fix Replay Viewer Auth Headers and Time Domain Mismatch

## Objective
Fix all integration failures in the replay viewer: add missing Authorization headers so session/reactions/chat load (REPLAY-04, REPLAY-06), and fix UTC ms vs relative ms mismatch so chat and reaction timelines synchronize correctly with video playback (REPLAY-07, REACT-09).

## Context
- `GET /sessions/:id` requires Cognito auth (api-stack.ts line 106-109)
- `GET /sessions/:id/reactions` requires Cognito auth (api-stack.ts line 286-289)
- `GET /sessions/:id/chat/messages` requires Cognito auth (api-stack.ts line 239-242)
- `authToken` in `ReplayViewer` is set asynchronously via `fetchAuthSession()` - session/reactions fetches fire before token is ready
- `SYNC_TIME_UPDATE` emits UTC epoch ms (e.g. 1709500000000), but `sessionRelativeTime` is elapsed ms since stream start (e.g. 5000) â€” comparison is always true

## Files to Change

### 1. `web/src/features/replay/ReplayViewer.tsx`
**REPLAY-04 + REPLAY-09**: Session fetch missing auth header and not gated on token.
**Reactions auth fix**: Reactions fetch missing auth header and not gated on token.

Changes:
- `fetchSession` useEffect: add `authToken` to dep array, guard with `if (!authToken) return`, add `Authorization: Bearer ${authToken}` header
- `fetchReactions` useEffect: add `authToken` to dep array, guard with `if (!authToken) return`, add `Authorization: Bearer ${authToken}` header

### 2. `web/src/features/replay/ReplayChat.tsx`
**REPLAY-06**: `useEffect` dep array missing `authToken`, initial fetch fires with empty token â†’ 401, no re-fetch when token arrives.

Changes:
- `useEffect([sessionId])` â†’ `useEffect([sessionId, authToken])` (line 52)
- Add `if (!authToken) return;` guard at start of `fetchMessages`

### 3. `web/src/features/replay/useReplayPlayer.ts`
**REPLAY-07 + REACT-09**: `SYNC_TIME_UPDATE` event passes UTC ms; `sessionRelativeTime` is relative ms since stream start. All messages/reactions are always visible (filter always true).

Fix: Use `player.getPosition() * 1000` (elapsed playback ms) instead of the raw UTC event `time` value.
