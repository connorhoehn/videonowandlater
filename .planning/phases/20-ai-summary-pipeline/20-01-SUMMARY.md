# Phase 20 Plan 01: AI Summary Pipeline Backend Summary

## One-Liner

Bedrock-triggered event pipeline with non-blocking error handling: invokes Claude Sonnet 4.5 on transcripts and stores summaries durably on session records without affecting transcript integrity.

## Completed Work

### Task 1: Extended Session domain model with AI summary fields

Added two new optional fields to the Session interface:

- **`aiSummary?: string`** — The one-paragraph summary text generated by Bedrock/Claude
- **`aiSummaryStatus?: 'pending' | 'available' | 'failed'`** — Pipeline state tracking for frontend display

Both fields are backward compatible (undefined for pre-Phase 20 sessions). Added comprehensive JSDoc comments for clarity.

**File**: `backend/src/domain/session.ts`

### Task 2: Created updateSessionAiSummary() repository function

Implemented a selective-update function that:

- Updates only `aiSummary` and/or `aiSummaryStatus` fields
- Never touches `transcriptText` field (critical safeguard on Bedrock/DynamoDB failures)
- Uses DynamoDB UpdateExpression with dynamic SET clauses
- Follows the non-blocking error pattern from `recording-ended.ts`
- Returns void on success, lets errors propagate to caller

**Test coverage**: 6 tests covering single-field updates, dual-field updates, field preservation, and error scenarios.

**Files**: `backend/src/repositories/session-repository.ts`, `backend/src/repositories/__tests__/session-repository.test.ts`

### Task 3: Created store-summary handler with Bedrock integration

Implemented an EventBridge-triggered Lambda handler (`store-summary.ts`) that:

1. **Extracts event details** from "Transcript Stored" EventBridge event (sessionId, transcriptText)
2. **Invokes Bedrock** using Claude Sonnet 4.5 Messages API with:
   - System prompt: "Generate a concise one-paragraph summary (2-3 sentences)..."
   - User prompt: Full transcript text
   - Max tokens: 500 (conservative for 2-3 sentence output)
3. **Stores summary** via updateSessionAiSummary() with aiSummaryStatus='available'
4. **Handles failures gracefully**:
   - Bedrock error → Sets aiSummaryStatus='failed' WITHOUT touching aiSummary (transcript preserved)
   - DynamoDB write error → Logs but doesn't throw (non-blocking pattern)

Uses TextDecoder to parse Bedrock's JSON response body.

**File**: `backend/src/handlers/store-summary.ts` (80 lines, well-commented)

### Task 4: Created comprehensive test suite for store-summary handler

Implemented 9 test cases covering:

1. ✓ **Successful invocation**: Bedrock succeeds → summary stored with aiSummaryStatus='available'
2. ✓ **Response parsing**: Correctly extracts summary text from responseBody.content[0].text
3. ✓ **Transcript preservation on Bedrock failure**: Error doesn't update aiSummary field
4. ✓ **Failure status setting**: Bedrock failure sets aiSummaryStatus='failed' explicitly
5. ✓ **Non-blocking storage failure**: Bedrock succeeds but DynamoDB fails → handler completes
6. ✓ **Double error handling**: Both Bedrock and DynamoDB fail → handler still completes
7. ✓ **Environment variable: model ID**: BEDROCK_MODEL_ID correctly passed to InvokeModelCommand
8. ✓ **Environment variable: region**: BEDROCK_REGION correctly passed to BedrockRuntimeClient
9. ✓ **Region fallback**: Falls back to AWS_REGION when BEDROCK_REGION not set

Mock setup properly isolates Bedrock and DynamoDB calls. All tests use jest.mock() for external dependencies.

**Files**: `backend/src/handlers/__tests__/store-summary.test.ts` (356 lines)

### Task 5: Wired StoreSummary Lambda and EventBridge rule in CDK

Updated `session-stack.ts` to create:

1. **StoreSummaryFn Lambda**:
   - Runtime: Node.js 20.x
   - Handler: store-summary.ts
   - **Timeout: 60 seconds** (critical for Bedrock latency 5-10s)
   - Environment variables: TABLE_NAME, BEDROCK_REGION, BEDROCK_MODEL_ID

2. **DynamoDB Permissions**: grantReadWriteData() for session table access

3. **Bedrock IAM Policy**:
   - Action: `bedrock:InvokeModel`
   - Resource: `arn:aws:bedrock:${region}::foundation-model/anthropic.claude-sonnet-4-5-20250929-v1:0`
   - Grants permission to invoke Claude Sonnet 4.5 specifically

4. **EventBridge Rule (TranscriptStoreRule)**:
   - Source: `custom.vnl`
   - Detail-type: `Transcript Stored` (emitted by Phase 19 store-transcript handler)
   - Targets: StoreSummaryFn

5. **Lambda Invoke Permission**: Explicit grant via addPermission() for EventBridge → Lambda invocation

TypeScript type checking passes with no errors.

**File**: `infra/lib/stacks/session-stack.ts` (49 new lines)
