```typescript
// Publish event to trigger Phase 19 transcription pipeline
const eventBridgeClient = new EventBridgeClient({ region: process.env.AWS_REGION });

try {
  await eventBridgeClient.send(
    new PutEventsCommand({
      Entries: [
        {
          Source: 'vnl.upload',
          DetailType: 'Upload Recording Available',
          Detail: JSON.stringify({
            sessionId,
            recordingHlsUrl,
          }),
          EventBusName: process.env.EVENT_BUS_NAME,
        },
      ],
    })
  );

  console.log(`Transcription pipeline triggered for session: ${sessionId}`);
} catch (error) {
  console.error(`Failed to publish transcription event for ${sessionId}:`, error);
  // Don't rethrow; session is already updated with HLS URL
  // Store-transcript handler can still work if it queries session.recordingStatus='available'
}
```

Key points:
- Import EventBridgeClient and PutEventsCommand from '@aws-sdk/client-eventbridge'
- Source: 'vnl.upload' (distinguishes uploaded video transcription from broadcast recording)
- DetailType: 'Upload Recording Available' (descriptive, matches pattern)
- Detail: { sessionId, recordingHlsUrl } (matches Phase 19 store-transcript consumer expectations)
- EventBusName: process.env.EVENT_BUS_NAME (must be passed via CDK environment variable)
- Non-blocking error handling: log but don't rethrow (session update already completed)
- Only publish on COMPLETE status, not on ERROR/CANCELED
