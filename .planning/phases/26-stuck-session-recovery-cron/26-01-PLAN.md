    1. Read env vars: `TABLE_NAME`, `AWS_REGION`, `MAX_RECOVERY_PER_RUN` (parseInt, default 25)
    2. Log pipeline stage entered
    3. Call `queryEndingSessions(tableName)` — queries STATUS#ENDING and STATUS#ENDED, returns merged Items[]
    4. Compute cutoff: `new Date(Date.now() - 45 * 60 * 1000).toISOString()`
    5. Filter items in-Lambda (endedAt threshold, transcriptStatus, recoveryAttemptCount cap)
    6. If filtered count > MAX_RECOVERY_PER_RUN, log warning and slice to MAX_RECOVERY_PER_RUN
    7. Process eligible sessions: `await Promise.all(eligibleSessions.map(item => recoverSession(item, tableName, awsRegion)))`

    9. Log pipeline stage completed with durationMs, sessionsRecovered, sessionsSkipped

    `queryEndingSessions` function: two sequential QueryCommands (ENDING then ENDED), spread into combined array. No Limit — expected small set. Catch errors per query (non-blocking, return []).

    `recoverSession` function:
    1. `logger.appendPersistentKeys({ sessionId })` (note: this mutates module-level logger, acceptable for Lambda sequential log correlation)
    2. Try UpdateCommand with conditional increment
    3. Catch ConditionalCheckFailedException — log warn "Concurrent cron race or cap already reached", return 'skipped'
    4. Extract new count from item (currentCount + 1)
    5. Try PutEventsCommand to default bus
    6. Catch PutEvents error — log error, return 'skipped' (don't double-count as recovered)
    7. Return 'recovered'
