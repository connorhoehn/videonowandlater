Create `web/src/features/upload/useHlsPlayer.ts` with this implementation:

Export a `Quality` interface:
```typescript
export interface Quality {
  level: number;   // -1 = Auto, 0+ = specific HLS level index
  label: string;   // "Auto", "1080p", "720p", etc.
  height: number;  // 0 for Auto
}
```

Export `useHlsPlayer(hlsUrl: string | undefined)` hook that:
1. Declares `videoRef = useRef<HTMLVideoElement>(null)`, `hlsRef = useRef<Hls | null>(null)`
2. State: `qualities: Quality[]`, `currentQuality: number` (default -1), `syncTime: number` (default 0), `isPlaying: boolean`, `isSafari: boolean`
3. `useEffect` triggered on `hlsUrl` changes:
   - Guard: `if (!hlsUrl || !videoRef.current) return`
   - If `Hls.isSupported()`:
     - `const hls = new Hls()` — store in `hlsRef.current`
     - `hls.attachMedia(videoRef.current)`
     - Register `Hls.Events.MANIFEST_PARSED` listener: build qualities array starting with `{ level: -1, label: 'Auto', height: 0 }`, then for each `data.levels[i]` push `{ level: i, label: lvl.height ? \`${lvl.height}p\` : \`Level ${i}\`, height: lvl.height || 0 }`. Call `setQualities(levels)`.
     - `hls.loadSource(hlsUrl)` — AFTER attachMedia, AFTER registering MANIFEST_PARSED listener
     - Register `timeupdate` on `videoRef.current`: `setSyncTime(video.currentTime * 1000)`
     - Register `play` listener: `setIsPlaying(true)`
     - Register `pause` listener: `setIsPlaying(false)`
     - Return cleanup: `hls.destroy(); hlsRef.current = null`
   - Else if `videoRef.current.canPlayType('application/vnd.apple.mpegurl')`:
     - `videoRef.current.src = hlsUrl`
     - `setIsSafari(true)`
     - No qualities populated (Safari has no quality API)
4. `setQuality(level: number)` callback:
   - `if (!hlsRef.current) return`
   - `hlsRef.current.nextLevel = level` — use nextLevel NOT currentLevel (currentLevel flushes buffer immediately causing stall)
   - `setCurrentQualityState(level)`
5. Returns: `{ videoRef, syncTime, isPlaying, qualities, currentQuality, setQuality, isSafari }`

Note: `syncTime` uses `currentTime * 1000` (milliseconds relative to video start) — consistent with the pattern from useReplayPlayer and needed by Phase 30 for comment anchoring.
