<objective>
Ship two standalone developer CLI scripts in a new `tools/` directory for pipeline introspection and replay.

`debug-pipeline.js` reads a DynamoDB session record and prints a human-readable pipeline status report — identity, recording state, all pipeline fields (transcriptStatus, aiSummaryStatus, mediaconvertJobId, etc.), and upload-specific fields.

`replay-pipeline.js` looks up the session in DynamoDB and then publishes the correct EventBridge event to resume the pipeline from any of four stages: `recording-ended`, `mediaconvert`, `transcribe`, `summary`.

Purpose: Give developers a fast local tool to diagnose stuck sessions and manually trigger pipeline recovery without navigating the AWS console.
Output: `tools/debug-pipeline.js`, `tools/replay-pipeline.js` — both runnable with `node` and no build step.
</objective>

function parseArgs() {
  const args = {};
  const argv = process.argv.slice(2);
  for (let i = 0; i < argv.length; i++) {
    if (argv[i].startsWith('--') && argv[i + 1] && !argv[i + 1].startsWith('--')) {
      args[argv[i].slice(2)] = argv[i + 1];
      i++;
    }
  }
  return args;
}

Create `tools/replay-pipeline.js` as a plain CommonJS script following the same structure as Task 1.

Script structure:
1. Shebang: `#!/usr/bin/env node`
2. Parse `--sessionId` and `--from` from argv. Also support optional `--table` override (default `vnl-sessions`).
3. Validate: if `--sessionId` or `--from` missing, print usage (show valid stages) and exit 1.
4. Valid stages: `recording-ended`, `mediaconvert`, `transcribe`, `summary`. If `--from` is not one of these, print `ERROR: Unknown stage '<stage>'. Valid stages: recording-ended, mediaconvert, transcribe, summary` and exit 1.
5. Initialize both `DynamoDBDocumentClient` (same pattern as debug tool) and `EventBridgeClient({ region })`. `region = process.env.AWS_REGION ?? 'us-east-1'`.
6. Fetch session via `GetCommand` — same pattern as Task 1. Exit 1 if not found.
7. Build the EventBridge event entry based on `--from`:

**`recording-ended`:**
- Check `session.recordingS3Path` — if absent, print `ERROR: Session has no recordingS3Path. Cannot publish Recording Recovery event.` and exit 1.
- Entry: `{ Source: 'custom.vnl', DetailType: 'Recording Recovery', Detail: JSON.stringify({ sessionId, recoveryAttempt: true, recoveryAttemptCount: 0, recordingHlsUrl: session.recordingHlsUrl, recordingS3Path: session.recordingS3Path }) }`
- Do NOT increment `recoveryAttemptCount` in DynamoDB — this is a developer tool and should not consume recovery slots.

**`mediaconvert`:**
- Entry: `{ Source: 'aws.mediaconvert', DetailType: 'MediaConvert Job State Change', Detail: JSON.stringify({ status: 'COMPLETE', jobId: session.mediaconvertJobId ?? 'manual-replay', userMetadata: { sessionId, phase: '19-transcription' }, outputGroupDetails: [{ outputDetails: [{ outputFilePaths: ['s3://vnl-transcription-vnl-session/' + sessionId + '/' + sessionId + 'recording.mp4'] }] }] }) }`

**`transcribe`:**
- Print warning: `NOTE: This event assumes transcript.json already exists at s3://vnl-transcription-vnl-session/<sessionId>/transcript.json. If not, transcribe-completed will fail with NoSuchKey.`
- Entry: `{ Source: 'aws.transcribe', DetailType: 'Transcribe Job State Change', Detail: JSON.stringify({ TranscriptionJobStatus: 'COMPLETED', TranscriptionJobName: 'vnl-' + sessionId + '-' + Date.now() }) }`

**`summary`:**
- Construct transcriptUri from `session.transcriptS3Path` if set, else `'s3://vnl-transcription-vnl-session/' + sessionId + '/transcript.json'`.
- Entry: `{ Source: 'custom.vnl', DetailType: 'Transcript Stored', Detail: JSON.stringify({ sessionId, transcriptS3Uri: transcriptUri }) }`

8. Send `PutEventsCommand({ Entries: [entry] })`.
9. Check `response.FailedEntryCount > 0` — if so, print error and exit 1.
10. Print:
```
Event published to EventBridge default bus.
  Stage:       <stage>
  Source:      <Source>
  Detail-Type: <DetailType>

NOTE: Lambda execution may take up to 20 seconds due to SQS buffering (Phase 31).
```
Exit 0.
