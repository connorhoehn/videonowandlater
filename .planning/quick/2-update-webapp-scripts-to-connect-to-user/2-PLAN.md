<context>
The webapp already loads AWS configuration from `web/public/aws-config.json` which is generated by the deploy script. We need to create helper scripts that verify this configuration exists and is valid before starting the development server.

Current flow:
1. `npm run deploy` generates `web/public/aws-config.json` from CDK outputs
2. `npm run dev` starts the Vite dev server
3. Frontend loads config and connects to AWS services

Missing: Verification step between deploy and dev to ensure configuration is present and valid.
</context>

<tasks>

<task type="auto">
  <name>Task 1: Create configuration verification script</name>
  <files>scripts/verify-config.sh</files>
  <action>
    Create a bash script that verifies the AWS configuration is properly set up for development.

    The script should:
    1. Check if `cdk-outputs.json` exists (indicates CDK deployment completed)
    2. Check if `web/public/aws-config.json` exists and is valid JSON
    3. Verify the config contains required fields: userPoolId, userPoolClientId, region, apiUrl
    4. Test connectivity to the API endpoint with a simple curl request
    5. Display the configuration status in a clear, formatted way
    6. Return exit code 0 if valid, 1 if invalid

    Make the script executable (chmod +x).
    Include helpful error messages that guide the developer to run `npm run deploy` if configuration is missing.
  </action>
  <verify>
    <automated>bash scripts/verify-config.sh && echo "Config verification script works"</automated>
  </verify>
  <done>Script exists, is executable, and correctly validates AWS configuration</done>
</task>

<task type="auto">
  <name>Task 2: Create development startup script</name>
  <files>scripts/dev.sh</files>
  <action>
    Create a bash script that wraps the development server startup with configuration checks.

    The script should:
    1. Call `./scripts/verify-config.sh` to validate configuration
    2. If config is missing or invalid:
       - Display a clear message explaining the issue
       - Ask if the user wants to run `npm run deploy` first
       - If yes, run deploy then continue; if no, exit
    3. If config is valid, display the configuration details:
       - API URL
       - User Pool ID
       - Region
    4. Start the Vite dev server with `cd web && npm run dev`
    5. Display the local dev server URL

    Make the script executable (chmod +x).
    Use clear formatting with colors if available (tput for terminal colors).
  </action>
  <verify>
    <automated>[ -f scripts/dev.sh ] && [ -x scripts/dev.sh ] && echo "Dev script created and executable"</automated>
  </verify>
  <done>Development script exists, is executable, and integrates config verification</done>
</task>
