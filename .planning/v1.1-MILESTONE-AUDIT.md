# v1.1 Milestone Audit: Replay, Reactions & Hangouts

**Milestone:** v1.1
**Audited:** 2026-03-04
**Status:** gaps_found
**Score:** 40/50 requirements satisfied

## Executive Summary

All 11 phases executed and individually verified. Two **P0 integration blockers** prevent core replay and hangout chat from working end-to-end:

1. **`get-session` strips recording fields** — `backend/src/services/session-service.ts` `getSession()` returns only `{sessionId, sessionType, status}`. `ReplayViewer.tsx` needs `recordingHlsUrl` to load the IVS player. Without it, every recording shows "Recording not available".

2. **HANGOUT sessions never transition to LIVE** — `join-hangout.ts` generates participant tokens but never calls `updateSessionStatus(LIVE)`. `send-message` checks `session.status !== LIVE` → HTTP 400. All hangout chat messages are silently dropped.

## Integration Gaps (by priority)

### P0: get-session strips recording fields (blocks all replay)

`backend/src/services/session-service.ts` `getSession()` returns only `{sessionId, sessionType, status}`.

`ReplayViewer.tsx` reads `session.recordingHlsUrl` (line 93, 205) — without it, IVS player never loads. Also reads `session.recordingDuration`, `session.userId`, `session.createdAt`, `session.endedAt`.

**Fix:** Extend `getSession()` response to include recording fields. Keep internal ARNs hidden (`claimedResources`, `stageArn`, `channelArn`) but expose `recordingHlsUrl`, `recordingDuration`, `thumbnailUrl`, `userId`, `createdAt`, `startedAt`, `endedAt`, `recordingStatus`.

### P0: HANGOUT sessions never transition to LIVE (blocks chat persistence)

`join-hangout.ts` generates Stage participant token but never calls `updateSessionStatus(tableName, sessionId, SessionStatus.LIVE)`.

`send-message` handler: `if (session.status !== SessionStatus.LIVE) return 400`. HANGOUT sessions stuck at `CREATING`.

**Fix:** Call `updateSessionStatus` with LIVE and set `startedAt` in `join-hangout.ts` after successful token generation.

### P1: Hangout recordings navigate to dead Stage (HANG-15 partial)

`RecordingFeed.tsx` navigates HANGOUT recordings to `/hangout/:id`. `HangoutPage` attempts to re-join a terminated Stage. User gets stuck.

**Fix:** Navigate HANGOUT recordings to `/replay/:id` (same replay viewer as broadcasts, which will work once get-session fix is in).

### P2: Remote participant labels are UUIDs (HANG-12 partial)

`join-hangout.ts` passes `attributes: { username }` to `CreateParticipantTokenCommand`. `useHangout.ts` reads `participant.attributes?.userId` — always undefined.

**Fix:** Add `userId` to attributes: `{ username, userId: username }`.
