## Integration Findings

### Critical (2)

**1. ReplayChat.tsx API path mismatch — breaks REPLAY-06, REPLAY-07**
- `web/src/features/replay/ReplayChat.tsx:26`
- Current: `fetch(\`${API_BASE_URL}/sessions/${sessionId}/messages\`)`
- Correct: `fetch(\`${API_BASE_URL}/sessions/${sessionId}/chat/messages\`)`
- Every replay page shows empty chat panel. Core replay experience broken.

**2. join-hangout response missing userId — degrades HANG-01, HANG-10**
- `backend/src/handlers/join-hangout.ts` response (lines 95-100)
- Current: Returns `{ token, participantId, expirationTime }`
- Correct: Add `userId: username` to response
- Local participant tile renders "undefined (You)".

### Warning (2)

**3. Dual EventBridge rules for recording-ended — affects REC-03, REC-05, REC-06**
- `infra/lib/stacks/session-stack.ts:187-196` (RecordingEndRuleV2) and `:292-303` (RecordingEndRule)
- Both match same IVS `Recording State Change` event. Handler invoked twice per recording end.
- Second invocation fails with DynamoDB optimistic lock version conflict (non-fatal, logged).

**4. recording-ended Stage ARN detection reads wrong field — breaks HANG-14, 15, 16**
- `backend/src/handlers/recording-ended.ts:31`
- Current: `const resourceArn = event.detail.channel_name`
- IVS RealTime Stage Recording End events use different field structure than broadcast channels.
- Hangout recording metadata never written → hangouts don't appear in home feed.

### Missing Feature (2)

**5. No hangout creation UI — HANG-02**
- `HomePage.tsx` only has `handleCreateBroadcast`. Hangout sessions unreachable from UI.

**6. No invite link generation — HANG-03**
- Likely deferred or interpreted as out-of-scope. No implementation in any phase.
