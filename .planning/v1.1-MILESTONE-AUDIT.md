## Critical Gap Details

### Gap 1 — Time Domain Mismatch (P1: Core replay feature broken)

**Affects:** REPLAY-07, REACT-09

The IVS Player `SYNC_TIME_UPDATE` event fires UTC wall-clock milliseconds (e.g. `1709500000000`). This value is stored in `syncTime` state via `useReplayPlayer`. Both `useSynchronizedChat` and `useReactionSync` then filter:

```typescript
msg.sessionRelativeTime <= currentSyncTime  // always true
```

`sessionRelativeTime` is ms-since-stream-start (e.g. `5000` = 5 seconds in). Comparing against UTC ms (≈1.7 trillion) means the filter is always `true` — all messages and reactions are visible from frame 0.

**Fix:** `const relativeMs = syncTime - session.startedAt; return msg.sessionRelativeTime <= relativeMs;` where `session.startedAt` is the epoch ms when the session went live.

**Files:**
- `web/src/features/replay/useSynchronizedChat.ts` line 25
- `web/src/features/replay/useReactionSync.ts`

---

### Gap 2 — ReplayViewer Missing Authorization Headers (P1: Video never loads)

**Affects:** REPLAY-04, REPLAY-06, REPLAY-09

`ReplayViewer.tsx` fetches two Cognito-protected endpoints without auth:
- `GET /sessions/:id` (line 63) — no `Authorization` header → 401 → `recordingHlsUrl`, metadata never load
- `GET /sessions/:id/reactions` (line 99) — no `Authorization` header → 401 → reactions never load

`ReplayChat.tsx` `useEffect` (line 53) also omits `authToken` from its deps array. Token is async, initial fetch fires empty → 401. No re-fetch when token arrives.

**Fix:** Add `Authorization: Bearer ${authToken}` to both ReplayViewer fetches; add `authToken` to ReplayChat `useEffect` deps and gate fetch on `authToken` being non-empty.

**Files:**
- `web/src/features/replay/ReplayViewer.tsx` lines 63, 99
- `web/src/features/replay/ReplayChat.tsx` line 53

---

### Gap 3 — Home Feed Includes Non-Available Recordings (P2: Data quality)

**Affects:** REPLAY-01

`getRecentRecordings` scans DynamoDB with `status='ended'` only. Sessions where recording is still `'processing'`, `'failed'`, or `'pending'` (e.g. sessions that ended before recording attached) appear in feed. UI renders "Awaiting recording..." overlays but they are permanently non-interactive.

**Fix:** Add `AND recordingStatus = :available` to the `FilterExpression` in `backend/src/repositories/session-repository.ts` line 218.

---

### Gap 4 — HangoutPage userId from localStorage (P3: Identity inconsistency)

**Affects:** HANG-13

`HangoutPage.tsx` lines 30-37 generates `user-<random>` from `localStorage` for `userId`. The IVS participant token uses Cognito `username`. Mismatch can affect chat moderator checks. Code comment acknowledges this as a placeholder. Fix: use `useAuth()` hook (same as `BroadcastPage.tsx`).
